获取授权认证凭证token
更新时间: 2019-11-15

1.通讯方式

服务器访问方式：http协议GET 及POST方法
​​​​​​​2.协议格式

Json协议
​​​​​​​3.接口地址

http://v1.chaoxing.com/openInter/getAccessToken?appid=&appkey=

参数名

参数说明

appid

【必传】身份唯一标识APPID

appkey

【必传】对接密匙APPKEY

​​​​​​​​​​​​​​4.返回信息说明

正确返回：

{

         "status":true成功,

         "data": {

"access_token": 授权认证凭证token

"expire_time": 授权认证凭证token超时时间

}

}

遇错返回：

{

         "status":  false失败,

                   "data": 失败说明

}

统一接口调用规则
更新时间: 2019-11-15

1.通讯方式

服务器访问方式：http协议GET 及Post方法
2.协议格式

Json协议
3.接口地址

http://v1.chaoxing.com/openInter/requestInter?appid=&access_token=

&expires_time=&routeId=  (¶meter1=¶meter2=.....)

接口说明：第三方开发者可以根据自己的业务需求从超星接口路由库中（下方有接口路由库列表及说明）选取不同的接口搭配进行调用；

注意：routeId是超星接口路由库中各接口的标识，当选择一个routeId时，需查看此routeId对应的接口说明及要求，并在调用时除以上四个参数外追加上调用此接口必须的参数，调用多个接口时只需更换routeId及追加routeId对应接口的参数即可

参数名

参数说明

appid

【必传】身份唯一标识APPID

access_token

【必传】授权认证凭证token（传递过程中须进行URLEncoder转译）

expires_time

【必传】授权认证凭证token超时时间（传递过程中须进行URLEncoder转译）

routeId

【必传】接口路由库中的各接口标识，利用此标识来确认请求目地接口，并返回此接口操作结果

parameter1

虚拟变动参数，为routeId对应接口的参数，具体参数名和要求请查看接口库列表说明

Parameter2

....

返回信息说明
正确返回：

{

         "status":  true为获取成功,

         "data":  routeId对应接口返回信息（具体可到接口库列表中查看说明）

}

遇错返回：

{

         "status":  false为获取失败,

                   "data": 失败说明

}
